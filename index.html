Git vocabulary

Open a term whenever Git's language feels confusing.

Commit+

An immutable project snapshot plus metadata and a link to its parent commit. Its ID is calculated from its contents.

Branch+

A movable name pointing to a commit. Creating a branch is lightweight; new commits move the checked-out branch forward.

HEAD+

Your current position. It normally points to the checked-out branch, which then points to a commit.

Index / stage+

The proposed snapshot for your next commit. git add copies selected file content into it.

origin+

The conventional short name for the remote you cloned or connected. It is only a label; it can have another name.

upstream+

The remote branch your local branch tracks, or—in a fork workflow—the original repository remote.

06 / GitHub workflow

From idea to pull request

Git manages history on your machine. GitHub helps people share that history, discuss it, and review it together.

01

Fork or clone

Get a local copy of the repository.

git clone <url>
02

Create a branch

Keep your change focused and isolated.

git switch -c feature/name
03

Commit & push

Publish your clean, meaningful commits.

git push -u origin feature/name
04

Open a pull request

Explain the why, link the issue, request review.

gh pr create
05

Review & merge

Discuss, improve, pass checks, then merge.

git switch main && git pull
Git vs. GitHub — what is the difference?

Git is the version-control tool that runs locally. GitHub is a hosting and collaboration platform for Git repositories. You can use Git without GitHub.
